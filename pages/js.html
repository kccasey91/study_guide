  • const: use all capitals if the const value is predetermined before runtime.

  • IMPORTANT: always give variables a clear and obvious name so you know what they do.

  • CamelCase if you have more than one word variables, stop being lazy!

  • Use new variables instead of reusing old ones!!

  • Convert non numbers to numbers with the unary +:
    let apples = "2";
    let oranges = "3";
    alert( +apples + +oranges ); // 5

  • Template literal: `Hello, ${variable or expression}`

  • Using back ticks(template literals) is more versatile and
    readable than using quotes for strings.
